# Issue #243 Part A — UI/UX Overhaul Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the seven owner-approved changes of issue #243 Part A — archived toggle → Filter popover, instant cluster-panel entrance, floating glass scenario card, and the two-line page-header redesign — plus the mandated CLAUDE.md / Mission Bento spec amendments.

**Architecture:** All changes are in `apps/web` (React 19 + Tailwind v4 CSS-first + Radix via the unified `radix-ui` package + `motion`). The cluster panel keeps its dialog/inert/focus contract; only its entrance motion, header anatomy, and scenario-pane surface change. The fleet console swaps a stranded toggle for a Popover-based Filter control. New `--glass-*` tokens live in `styles.css` beside the existing per-theme tokens.

**Tech Stack:** React 19, TanStack Router `Link`, Radix Popover/Tooltip (already wrapped in `ui/`), `motion` 12 (`m.*` from `motion/react-m`), Tailwind v4, Vitest + RTL (jsdom), Playwright.

## Global Constraints

- Branch `feat/243-ui-overhaul` off `origin/dev`; PR → `dev`. Commit style `feat(web): …` / `docs: …`.
- Strict TS (`noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`); no `any`, no suppressions.
- Both themes for every visual change; status never color-alone (WCAG 1.4.1); focus ring is global `:focus-visible` (never suppress).
- Motion: never animate blur radius; transform/opacity only; global reduced-motion kill-switch + `MotionConfig reducedMotion="user"` stay authoritative.
- Sync-sensitive constants stay: `SCENARIO_PANE_WIDTH=340`, `PANE_SIDE_BY_SIDE_QUERY='(min-width: 1024px)'`, `.cluster-panel{width:100vw}`, `RESIZE_DEBOUNCE_MS=300` (still outlasts the pane's 280 ms animation).
- `pnpm` is invoked as `npx -y pnpm@11 …` on this machine. Unit tests: `npx -y pnpm@11 --filter @lcm/web test`. Dev server: `DATABASE_URL='postgresql://lcm:lcm@localhost:5432/lcm' PORT=8090 npx -y pnpm@11 dev`.
- The chip-for-crit caveat (issue item 4) is resolved during live review: default is chip-only; if `ORDER NOW` reads too quiet in the live check, fall back to chip + full-width crit-only banner below the header.
