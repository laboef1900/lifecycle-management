> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the four High-severity findings from the #243 Part B audit backlog (`docs/uiux-audit-2026-07.md`) — all small, all real WCAG/legibility failures.

**Architecture:** Four independent, surgical fixes in `apps/web` only — two visual primitives (BulletMeter ticks, gantt SVG labels) get the codebase's established surface-halo treatment; four hover classes swap amber brand fill for the ghost pattern; one behavior change closes the sub-`lg` scenario sheet after a successful Apply/Clear so the chart it edits becomes visible. No API, schema, or token changes.

**Tech Stack:** React 19, Tailwind 4 (CSS-first tokens), Vitest + RTL (colocated), Playwright e2e (`apps/web/playwright/`), framer-motion (`m.*` + AnimatePresence).

## Global Constraints

- Both themes MUST work for every styling change; in dark theme `--warning` and `--accent` are the identical hex `#ffc53d` (CLAUDE.md house style) — that overlap is a recorded decision, do NOT touch tokens.
- Status/semantic tokens only — never raw palette classes; color MUST NOT be the only channel (WCAG 1.4.1).
- Strict TS (`noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`); no `any`, no suppressions.
- Prettier: single quotes, semicolons, width 100 (lint-staged formats on commit).
- Worktree-based flow: branch `fix/243-part-b-high` off `origin/dev` in `../243-part-b-high`; PR targets `dev`; commits `fix(web): …`.
- The panel/pane geometry constants are sync-sensitive (`SCENARIO_PANE_WIDTH=340`, `PANE_SIDE_BY_SIDE_QUERY` 1024px) — this plan does not move them.
- Halo color is `var(--card)` — the exact pattern `cluster-tile-chart.tsx` already uses for labels crossing marks (spec §6).
- Local Playwright needs the API started with `RATE_LIMIT_MAX=2000` (rate-limit 429s masquerade as login redirects otherwise).

### Task 1: BulletMeter — threshold ticks survive any fill; crit differs by shape

**Files:**

- Modify: `apps/web/src/components/fleet/bullet-meter.tsx:47-58`

**Why:** In dark theme the warn tick (`bg-warning/70`) sits on the accent-derived amber fill once `value >= warn` — identical hex at 0.7 alpha ⇒ the tick disappears exactly when it matters. Warn vs crit also differ by hue alone (WCAG 1.4.1).

```tsx
// Below `lg` the Scenario sheet covers the very chart a scenario edits
// (#243 Part B): a successful Apply/Clear closes the sheet so the user
// lands on the updated forecast with the header indicator visible. At
// `lg`+ the chart updates live beside the pane, so it stays open. Guarded
// on `paneOpen` so a change landing mid-exit cannot re-dispatch 'close'.
const paneCoversContent = paneLayout.coversContent;
const handleScenarioChange = useCallback(
  (next: ScenarioWire | null): void => {
    setScenario(next);
    setAnnouncementOverride(
      next ? `Scenario active: ${describeScenario(next)}.` : 'Baseline forecast restored.',
    );
    if (paneCoversContent && paneOpen) closePane();
  },
  [paneCoversContent, paneOpen, closePane],
);
```
